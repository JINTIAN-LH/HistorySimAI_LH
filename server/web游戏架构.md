# 网页游戏架构基线

## 项目定位

当前项目属于浏览器文字策略模拟游戏，核心体验是回合推进、叙事反馈、数值结算和多系统联动，不需要引入实时联机或重型渲染引擎。

## 架构选择

- 推荐类型：浏览器文字策略模拟
- 推荐前端外壳：React 壳层承载导航、配置门禁和页面骨架
- 推荐渲染层：DOM 叙事渲染，保留旧有 `js/ui` 模块输出能力
- 推荐核心层：系统驱动的状态机与回合结算逻辑
- 推荐数据层：本地存档 + `fetch` + Express 代理

## 渐进迁移原则

- 先建立 `client/server` 启动壳，再逐步迁移旧页面，不直接重写全部玩法模块
- 旧 `js/systems`、`js/ui`、`js/storage` 逻辑优先保持兼容
- 世界观迁移只替换语义皮层，不破坏回合循环和存档结构
- 所有架构调整必须可以被 `npm run build`、`npm test` 和 `npm run verify:experience` 验证

## 约束

- 不为了形式统一而引入 Phaser、实时联机或 ECS 重写
- 不拆散现有存档链路与季度结算链路
- 所有新壳层都必须允许旧玩法逻辑继续挂载和运行
