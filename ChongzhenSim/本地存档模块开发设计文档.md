# AI友好型本地存档模块开发设计文档
## 文档版本
- 版本：V1.0
- 日期：2026.04.04
- 适用场景：**崇祯模拟器/君主模拟/文字策略类单机游戏** 本地存档、读档、自动备份、AI交互适配

## 一、模块定位与核心目标
### 1. 模块定位
本模块是游戏**核心数据持久化组件**，独立于游戏逻辑、UI、AI决策模块，负责**游戏进度的本地存储、读取、校验、备份、清理**，为游戏本体与AI决策系统提供稳定、可解析、无歧义的存档数据。

### 2. 核心目标
1. **纯本地运行**：无云端依赖，离线可用，适配单机游戏特性
2. **AI友好**：数据结构标准化、无冗余、可直接解析，支持AI快速读取/写入游戏状态
3. **高稳定性**：自动备份、损坏校验、回滚机制，杜绝存档丢失
4. **轻量高效**：读写速度≤100ms，占用空间小，适配大量数值类模拟游戏
5. **易扩展**：支持后续新增属性、剧情节点、MOD适配

### 1. 功能需求

|---------|---------|

- **安全性**：基础防篡改校验，不强制加密（保证AI可访问）

## 三、总体架构设计
### 1. 架构分层（极简、解耦、AI友好）
```
游戏上层逻辑（UI/剧情/数值/AI）
        ↑ ↓
=========================
  存档核心模块（4层）
=========================
1. 对外接口层 → 给游戏/AI调用的统一方法
2. 数据处理层 → 数据校验、版本升级、格式化
3. 文件操作层 → 本地文件读写、备份、删除
4. 配置常量层 → 路径、规则、版本、命名规范
```

### 2. 模块边界
- **不侵入**：不修改游戏业务逻辑，只负责数据读写
- **单向依赖**：游戏/AI可调用存档模块，模块不依赖游戏其他代码
- **数据透明**：除校验字段外，所有数据明文可查（AI友好核心）

## 四、数据结构设计（AI友好核心）
### 1. 存档文件规范
- **格式**：`.json`（明文、标准、AI原生支持）
- **命名**：`save_时间戳_槽位号.json`
  示例：`save_1746412345_01.json`
- **单文件独立**：一个存档=一个文件，不合并、不嵌套数据库

### 2. 顶级数据结构（固定结构，AI可直接映射）
```json
{
  "save_version": "1.0",
  "save_id": "save_1746412345_01",
  "timestamp": 1746412345,
  "game_time": "崇祯八年-三月-初七",
  "player_progress": "主线：平台整顿",
  "checksum": "a1b2c3d4e5f6",
  "game_data": {},
  "ai_visible_data": {}
}
```

### 3. 核心字段说明
| 字段 | 用途 | AI友好性 |
|------|------|---------|
| save_version | 存档版本，用于旧档升级 | 固定键名，AI可判断兼容性 |
| save_id | 唯一存档标识 | 无歧义，用于AI定位文件 |
| timestamp | 时间戳（秒） | 机器可读，用于排序/筛选 |
| game_time | 游戏内时间 | 人类+AI双可读 |
| player_progress | 游戏进度摘要 | AI快速判断存档阶段 |
| checksum | 数据校验码 | 防损坏，AI可验证数据有效性 |
| game_data | 游戏核心数据 | 标准化结构，无动态键名 |
| ai_visible_data | AI专用数据区 | 独立隔离，AI无需过滤无关字段 |

### 4. game_data 标准结构（崇祯模拟器适配）
```json
"game_data": {
  "player": {
    "name": "朱由检",
    "health": 85,
    "morale": 70,
    "authority": 90
  },
  "nation": {
    "money": 1250000,
    "food": 890000,
    "army_num": 350000,
    "popular_feel": 60
  },
  "officials": [],
  "events": [],
  "flags": {
    "has_opened_beijing_troop": true,
    "has_signed_peace": false
  }
}
```

### 5. AI专用数据区（ai_visible_data）
**专为AI决策设计**：仅存放AI需要判断的关键数据，无UI、渲染、音效等冗余字段。
```json
"ai_visible_data": {
  "danger_level": 3,
  "treasury_rate": -5,
  "enemy_strength": 75,
  "available_decisions": ["调兵", "加税", "赈灾"]
}
```

## 五、存档文件存储规则
### 1. 存储路径（跨平台通用）
- Windows：`C:/Users/用户名/AppData/Local/ChongzhenSimulator/Saves/`
- 统一对外暴露：`游戏根目录/saves/`（快捷方式指向真实路径）

### 2. 目录结构
```
Saves/
├─ auto/              # 自动存档
│  ├─ auto_1.json
│  └─ auto_2.json
├─ backup/            # 损坏前自动备份
│  └─ save_xxx.bak
├─ save_01.json       # 手动槽位1
├─ save_02.json       # 手动槽位2
└─ quick_save.json    # 快速存档
```

### 3. 存档数量限制
- 手动存档：最多20个
- 自动存档：最多10个（循环覆盖）
- 备份存档：最多5个（按时间保留最新）

## 六、核心功能流程设计
### 1. 存档流程
1. 游戏/AI传入标准化数据
2. 模块生成唯一save_id、时间戳
3. 填充ai_visible_data专用区
4. 生成校验码checksum
5. 写入JSON文件
6. 自动清理超量旧档
7. 返回成功/失败状态

### 2. 读档流程
1. 传入存档ID/路径
2. 检查文件是否存在
3. 读取并解析JSON
4. 校验checksum判断是否损坏
5. 版本升级（旧档→新版结构）
6. 返回完整数据给游戏/AI
7. 损坏则自动加载备份

### 3. 自动存档流程
- 触发条件：
  1. 每游戏1天自动存1次
  2. 进入关键剧情前自动存
  3. 退出游戏前强制存
- 规则：自动档循环覆盖，保留最近10个

### 4. 损坏修复流程
1. 读档校验失败
2. 提示：存档已损坏，正在恢复备份
3. 自动加载backup目录下最近备份
4. 恢复成功后覆盖原损坏文件

## 七、AI友好特性设计（核心亮点）
### 1. 数据层面
- **纯JSON明文**：无二进制、无复杂加密，AI可直接读写
- **固定键名**：不使用动态字符串、不使用数字作为键名
- **独立AI数据区**：`ai_visible_data` 无冗余，AI可直接读取决策
- **强类型规范**：数字=数字，布尔=布尔，无混合类型

### 2. 接口层面
- 提供**AI专用读写接口**：无需游戏UI，直接操作数据
- 支持**批量获取存档列表**：AI可批量分析历史进度
- 支持**快速存档**：AI决策后可10ms内保存当前状态

### 3. 兼容层面
- 存档版本升级**不破坏AI可读结构**
- 新增字段自动补默认值，不抛出解析错误
- 支持AI传入自定义字段并持久化（AI训练/调试用）

## 九、数据安全与校验
### 1. 校验方式
- 采用**MD5轻量校验**：
  `checksum = MD5( game_data + ai_visible_data + save_version )`
- 不加密数据，仅校验完整性（AI友好）

### 2. 防篡改
- 校验失败即判定为损坏/篡改，自动启用备份
- 不强制反作弊，不干扰AI修改存档（用于调试/创作）

### 3. 备份机制
- 每次覆盖存档前自动备份
- 备份文件格式：`.bak`，保留5份

## 十、版本升级兼容设计
### 1. 旧存档升级规则
- 游戏版本更新后，存档结构变化时：
  1. 读取旧存档`save_version`
  2. 通过升级函数补全新增字段
  3. 保留所有旧字段（AI兼容）
  4. 写入新版本存档

### 2. 永不删除旧键名
- 废弃字段仅隐藏，不删除，保证AI脚本兼容

## 十二、异常处理
| 异常场景 | 处理方式 |
|---------|---------|
| 文件不存在 | 返回null，提示无存档 |
| JSON解析失败 | 判定损坏，自动恢复备份 |
| 权限不足 | 提示目录不可写，切换到桌面临时目录 |
| 磁盘空间不足 | 终止写入，返回错误提示 |
| 版本过低 | 自动升级数据结构 |
