# 骑马与砍杀2核心注意力消耗玩法模块开发文档（适配崇祯模拟器融合）
## 一、模块定位与核心目标
### 1.1 模块定位
本模块提取**骑马与砍杀2（Bannerlord）中高注意力消耗、强操作/决策压力的核心玩法**，剥离原版开放世界、大地图经营等冗余内容，封装为**独立可插拔的「战场指挥+即时战术+近身格斗」复合战斗模块**，适配《崇祯模拟器》的「内政-军事-危机」核心循环，作为游戏中**大规模野战、城防攻防、精锐小队突袭**的核心战斗子系统，替代传统文字/简化战报，提升玩家沉浸与决策压力。

### 1.2 核心目标
1.  复刻骑砍2核心注意力消耗点：**实时阵型操控、多兵种协同、主将身临前线、动态战场应变、资源/士气/伤亡即时权衡**
2.  适配崇祯模拟器：保留明朝军事体系（卫所、边军、车营、火器、骑兵）、战损影响全局（兵力、粮草、军心、声望）、战斗结果直接反哺内政/外交
3.  AI友好：**结构化参数、标准化接口、可配置规则、无硬编码逻辑、支持数值/玩法快速迭代**

## 二、核心注意力消耗玩法拆解（骑砍2核心）
### 2.1 核心注意力维度（玩家持续聚焦点）
| 注意力类型 | 核心玩法机制 | 消耗强度 | 适配崇祯场景 |
|------------|--------------|----------|--------------|
| 实时战术操控 | 阵型切换（方阵/楔形/圆阵/散阵）、阵线拉扯、侧翼包抄、预备队调度、骑兵冲锋时机 | 极高 | 辽东野战、农民军围剿、京师保卫战 |
| 主将临场决策 | 亲自带队冲锋、切换武器/马具、格挡/劈砍/突刺微操、救援友军、斩杀敌将、负伤/落马风险 | 高 | 精锐突击、主将督战、斩首行动 |
| 多兵种协同 | 步兵（长枪/刀牌）抗线、弓箭手/火铳手压制、骑兵（关宁铁骑/蒙古骑兵）迂回、车营/火炮掩护、兵种克制（长枪克骑兵、火铳克步兵） | 极高 | 步骑炮协同、车营防御、边军混编作战 |

### 2.2 核心玩法闭环（注意力流设计）
1. 战前准备（低注意力，铺垫压力）：**配置兵力（兵种比例、人数）、军械（火器/弓箭/马匹）、阵型预设、主将装备/技能、地形选择**
2. 战场开局（注意力快速拉升）：**双方列阵、接敌、远程火力对射、骑兵试探**
3. 中期决战（注意力峰值）：**阵线拉扯、侧翼突破、预备队投入、主将冲锋、士气波动、局部绞杀**
4. 后期收尾（注意力持续紧绷）：**追击/撤退、溃兵收拢、伤员统计、军械清点、防止反杀**
5. 战后结算（注意力回落，反馈全局）：**战损上报、粮草消耗、军心变化、声望/爵位变动、触发内政事件（如朝廷嘉奖/追责、地方叛乱）**

## 三、模块架构设计（AI友好，可插拔）
### 3.1 模块分层结构（标准化接口）
#### 3.1.1 上层接口层（对接崇祯模拟器主程序）
- 输入接口（主程序→战斗模块）：
    1.  战场参数：**战场类型（野战/攻城/守城/突袭）、地形（平原/山地/城池/雪地）、天气、时间（昼/夜）**
    2.  双方兵力：明军（玩家）：兵种ID、数量、等级、士气、军械、主将属性/技能；敌军（后金/农民军/叛军）：同上
    3.  全局约束：**粮草上限、援军抵达时间、战损容忍度、朝廷要求（必须全歼/守住城池/限时取胜）**
- 输出接口（战斗模块→主程序）：
    1.  战斗结果：**胜/负/平、歼敌数、我方伤亡（死亡/重伤/轻伤）、俘虏数、军械损耗、粮草消耗**
    2.  状态反馈：**我方军心变化、主将声望/爵位、触发事件（如敌将阵亡、我方兵变、援军赶到）**
    3.  数据存档：**战斗日志、阵型操作记录、决策节点，用于AI复盘/学习**

#### 3.1.2 核心逻辑层（注意力消耗核心，AI可配置）
1.  阵型系统：**预设10种明朝经典阵型（长枪方阵、车营圆阵、骑兵楔形阵、火铳三段阵等），支持实时切换/微调，参数化定义阵型宽度、密度、移动速度、防御/攻击加成**
2.  兵种系统：**明朝标准化兵种（卫所步兵、边军长枪兵、关宁铁骑、神机营火铳手、车营炮兵、蒙古附庸骑兵），参数化克制关系、攻击/防御/士气/移动属性**
3.  主将系统：**主将属性（武力、统御、勇气、智谋）、技能（冲锋、格挡、鼓舞、侦查）、微操动作（攻击/格挡/骑马/下马/换武器）、负伤/落马机制（影响指挥效率）**
4.  士气系统：**实时计算士气值（受伤亡、阵型、主将状态、地形、天气影响），士气阈值触发：坚守/进攻/溃逃/投降，局部溃败扩散机制**
5.  战场物理系统：**地形移动惩罚、远程弹道（弓箭/火铳/火炮）、碰撞检测、冲锋伤害加成、骑兵冲垮步兵机制**

#### 3.1.3 表现层（轻量化，适配模拟器）
- 简化2D/伪3D战场可视化：**阵型色块、兵种图标、主将标记、士气条、伤亡数字、远程弹道特效**
- 操作UI：**阵型快捷按钮、兵种分组、预备队面板、主将技能快捷键、战场小地图**
- 注意力提示：**关键预警（侧翼被包、士气暴跌、箭矢耗尽、主将负伤），强制玩家聚焦核心决策**

## 四、核心玩法规则（注意力消耗细节，AI可执行）
### 4.1 实时阵型操控规则（核心注意力点）
1.  玩家可将部队分为**3-5个独立编队**（步兵、骑兵、火铳、炮兵、预备队），每个编队独立操控阵型、移动、攻击目标
2.  阵型切换有**冷却时间（2-5秒）**，切换期间阵型脆弱、防御下降，强制玩家预判战场，避免频繁操作
3.  移动速度与阵型密度正相关：**密集阵型（方阵）防御高、移动慢；松散阵型（散阵）移动快、防御低**，玩家需在抗线与机动间权衡
4.  侧翼保护：**阵型侧翼无掩护时，防御下降50%，极易被骑兵包抄崩溃**，玩家必须持续关注阵线完整性

### 4.2 多兵种协同规则（注意力分散压力）
1.  基础协同逻辑：**步兵抗线→远程压制→骑兵迂回包抄→预备队补缺口**，任一环节失误导致全线崩溃
2.  火器特殊规则：**神机营火铳有装填时间（5秒）、射程有限、雨天失效，需与弓箭手交替射击，保持持续压制**
3.  车营规则：**战车围成圆阵，提供防御掩体，火铳/火炮在车内射击，移动缓慢，适合守城/防御战**
4.  骑兵规则：**关宁铁骑冲锋有冷却，冲垮步兵后需重整，不可持续冲锋，需把握时机（敌阵混乱时）**

### 4.3 主将临场规则（微操+决策双重压力）
1.  主将可选择**坐镇指挥（提升全军士气+15%，无负伤风险）** 或 **亲自冲锋（提升编队士气+30%，可斩杀敌将，但有20%/次负伤概率，负伤后指挥效率下降50%）**
2.  主将微操：**格挡（消耗注意力，抵消伤害）、劈砍/突刺（高伤害，需瞄准）、骑马突围（被包围时可用）、鼓舞（消耗精力，瞬间提升编队士气）**
3.  主将阵亡：**全军士气暴跌-50%，大概率全线溃败，战斗直接失败**，强制玩家保护主将

### 4.4 动态战场应变规则（不可预测性，持续消耗注意力）
1.  士气波动：**每损失10%兵力，士气-15；每歼灭10%敌军，士气+10；主将在场+20；地形劣势-10**
2.  突发事件（随机触发，AI可配置概率）：
    - 局部溃败：某编队士气崩溃，向友军扩散
    - 援军抵达：我方/敌方援军在战斗5-10分钟后抵达，改变战局
    - 天气突变：雨雪导致火器失效、骑兵移动下降
    - 粮草/箭矢耗尽：远程部队无法攻击，步兵士气下降
3.  地形约束：**山地骑兵速度-40%，适合步兵伏击；平原适合骑兵冲锋；城池攻防有城墙/城门/云梯机制**

### 4.5 战损与全局联动规则（注意力延伸，影响长期决策）
1.  兵力不可逆：**死亡士兵无法恢复，重伤士兵需30天+粮草救治，轻伤士兵7天恢复**
2.  军械损耗：**弓箭/火药/马匹消耗计入全局资源，战后需从国库/地方补给**
3.  军心/声望：**大胜→军心+50、声望+20、朝廷嘉奖；大败→军心-80、声望-30、可能触发朝廷追责/地方叛乱**
4.  俘虏处理：**可收编（消耗粮草，提升兵力）、斩杀（提升威慑，降低敌方士气，但影响民心）、释放（提升外交好感）**

## 五、模块接入崇祯模拟器流程（可执行）
### 5.1 接入步骤
1.  主程序触发战斗：**当玩家选择「出征」「守城」「突袭」时，调用战斗模块接口，传入战场参数、双方兵力、全局约束**
2.  模块初始化：**加载兵种/阵型/事件配置，初始化战场、编队、主将、士气**
3.  战斗执行：**玩家实时操控阵型、兵种、主将，模块实时计算伤害、士气、战损、触发事件**
4.  战斗结束：**模块返回结果、状态、数据，主程序更新全局数据（兵力、粮草、军心、声望），触发后续内政/外交事件**
5.  存档/复盘：**战斗日志存入存档，支持AI复盘、玩法优化**

### 5.2 冲突适配（解决骑砍与模拟器差异）
1.  兵力规模：**骑砍小兵团→模拟器大规模（数千人），模块支持兵力缩放，阵型/士气逻辑不变**
2.  明朝军事特色：**替换骑砍兵种为明朝体系，加入车营、神机营、卫所等特色单位**
3.  全局影响：**战斗结果直接绑定模拟器核心数值（国库、民心、军心、疆域），无独立闭环**

## 六、AI开发与迭代规范（核心要求）
### 6.1 AI友好设计要点
1.  全参数配置化：**所有数值（属性、加成、概率、冷却）均在配置文件中，无硬编码，支持AI批量修改/优化**
2.  规则标准化：**所有玩法逻辑（克制、士气、阵型）均为条件-触发-效果的标准化格式，AI可解析/生成/调整**
3.  数据可采集：**战斗过程中记录所有玩家操作、决策、结果，形成数据集，用于AI训练/强化学习**
4.  接口轻量化：**输入输出均为JSON格式，无复杂依赖，支持AI快速接入/测试**

### 6.2 迭代流程
1.  基础版本：实现核心阵型、兵种、主将、士气系统，完成基础野战
2.  扩展版本：加入攻城/守城、车营、火器、援军、天气机制
3.  优化版本：基于战斗数据，AI调整数值（士气阈值、兵种克制、阵型加成），平衡注意力消耗与游戏体验
4.  融合版本：深度对接崇祯模拟器内政/外交/危机系统，形成完整军事循环
